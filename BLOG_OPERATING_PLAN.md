# Cellbedell Blog 工作主軸

這個專案分成兩條主線進行：

1. 建立並維護部落格網站
2. 針對 VivaTech 2026 進行每日追蹤、發文與部署

`BLOG_POSITIONING.md` 定義 blog 的長期定位與編輯風格；本文件定義實際執行方式。

內容後台使用 Google Sheets：

https://docs.google.com/spreadsheets/d/1f5pOv6S8H-cJ1IWsq2kOU4dqJnqvDfWq8m5a_IDuIYg

## 1. 建立部落格網站

### 目標

把 Cellbedell Blog 做成一個可長期擴充的靜態多頁網站，而不是只服務單一文章或單一活動。

### 目前網站結構

- `index.html`：部落格首頁，放最新文章、系列入口、每日更新清單。
- `series-vivatech-2026.html`：VivaTech 2026 系列總覽頁。
- `posts/`：每日文章與系列文章存放位置。
- `assets/`：文章圖片、主視覺與引用圖片存放位置。
- `styles.css`：全站共用視覺樣式。
- `BLOG_POSITIONING.md`：長期編輯定位與系列規則。
- `CONTENT_BACKEND.md`：Google Sheets 內容後台欄位與每日發文規則。

### 網站維護原則

- 首頁要能讓讀者快速理解目前正在更新哪些系列。
- 每個系列都應該有自己的系列總覽頁。
- 每篇文章都要能從首頁或系列頁被找到。
- 圖片檔案要放進 `assets/`，不要散落在根目錄。
- 文章檔名使用 `posts/YYYY-MM-DD-topic.html`。

### 未來系列方向

- VivaTech / 科技展觀察
- 國際科技媒體觀察
- 旅館生活科技
- AI 與人文
- 時尚與生活

## 2. VivaTech 2026 每日追蹤與發文

### 目標

在 VivaTech 2026 展會期間，每天整理最新資訊，產出一篇可發布的繁體中文 blog 文章。文章不是單純新聞整理，而是要符合 Cellbedell Blog 的風格：科技、設計、時尚、新創、生活方式與未來城市的混合觀察。

### 每日流程

1. 檢查 VivaTech 官方網站、官方社群、press release 與活動頁。
2. 補充 2 到 4 個可靠媒體或參展品牌來源。
3. 將今日文章列與來源填入 Google Sheets 內容後台。
4. 選出今日主題，例如 AI、法國新創、健康科技、永續移動、創意產業。
5. 找 1 到 3 張可用圖片，並記錄圖片來源。
6. 產出文章大綱。
7. 產出繁體中文草稿。
8. 使用者確認後，新增文章頁並更新首頁與系列頁。
9. 檢查連結與圖片路徑，部署到 Netlify。

### 每日文章格式

- 主標題
- 副標或導言
- 發布日期
- 系列名稱：VivaTech / 科技展觀察
- Tags
- 主圖與圖片來源
- 3 到 5 個重點段落
- 台灣讀者或生活場景觀察
- 參考來源
- 下一篇可延伸方向

### 目前進度

- `2026-06-19`：已完成系列首篇總覽文。
- `2026-06-20`：預計整理展會最後一天觀察，聚焦 AI、法國新創與展後值得追的生活科技。

### 自動提醒

已建立每日提醒 automation：`vivatech-2026`，每天台灣時間上午 `10:00` 喚醒此 thread。

提醒目的：

- 在展會期間喚醒此 thread。
- 提醒先整理來源、選題、圖片與草稿方向。
- 採「草稿確認後部署」流程，不直接自動改網站或部署。

## 3. 部署與上線

### 主要平台

主要部署平台使用 `Netlify`。網站是純靜態站，不需要 build step；Publish directory 使用 `.`。

### 每次部署前

- 確認首頁最新文章在上方可第一時間看到。
- 確認系列分類入口保留在首頁下方。
- 確認當日文章包含標題、副標、日期、tags、圖片來源與參考來源。
- 執行 `npm run check:links`，確認本機連結與圖片路徑不缺檔。

### 每次部署後

- 檢查 Netlify 首頁可開。
- 檢查 VivaTech 系列頁可開。
- 檢查最新文章頁可開。
- 檢查圖片正常顯示，行動版不溢出。

## 執行順序

短期優先順序：

1. 保持首頁、系列頁與文章頁可以正常瀏覽。
2. 每天完成 VivaTech 2026 一篇更新，確認後部署。
3. 每篇文章都補齊圖片與來源註記。
4. 展會結束後，將 VivaTech 系列整理成完整回顧。

中期優先順序：

1. 將首頁從 VivaTech 專題入口擴充成多系列入口。
2. 為國際科技媒體觀察、旅館生活科技、AI 與人文、時尚與生活建立系列頁。
3. 建立文章模板，降低每日新增文章的重複工作。
